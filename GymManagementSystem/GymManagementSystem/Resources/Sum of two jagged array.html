      Finally i did it thanks folks

      the code is :

      Code Snippet

      public static int[][] Union(int[][] a0, int[][] a1)
              {
                  if (a0 == null) a0 = new int[0][];
                  if (a1 == null) a1 = new int[0][];
                  int[][] union = new int[a0.Length + a1.Length][];
                  for (int i = 0; i < a0.Length; i++)
                  {
                      union[i] = a0[i];
                  }
                  int j = 0;
                  for (int i = a0.Length; i < union.Length; i++)
                  {
                      union[i] = a1[j++];
                  }
                  return union;
              }

      Well, the biggest issue with performing a sum or union to a jagged array is that you'll have to define what happens in each row, you can run into a case where the lengths are the same, or the lengths are different. What do you do in those cases... in a simple join operation it's easy so long as you want to keep each row joined with the same row.

      Code Snippet

      int[][] union = ...;
      ...
      for( int i = 0; 0 < rows; i++ )
      {
      union[i] = new int[ length1 + length2 ];
      Array.Copy( a0[i], union[i], a0[i].Length );
      Array.Copy( a1[i], 0, union[i], a0[i].Length, a1[i].Length );
      }
      return union;

      excuse the psudocode, but it's the general idea behind it.

      However you have to make sure that each jagged array are of the same number of rows (or handle the case where they are not)

      Now doing sums of the values are harder, what if your jagged arrays look like:

      [ 1, 4, 5, 12, 3 ]
      [ 4 ]
      [ 67, 75, 9 ]
      [ 2, 99 ]

      and

      [ 5 ]
      [ 63, 12, 87, 3, 23 ]
      [ 45, 7 ]
      [ 8 ]

      now they have the same number of elements, but vastly different structures, do you assume zero for any place that isn't defined? do you need to normalize the arrays so they match? do you only want matching jagged arrays?

      the main thing is you just need to loop your dealing with a single array over the array of arrays. This is different from dealing with a multi-dimentional arrays, which still need to consider different sizes but there are well defined matrix calculations on md arrays for operations as such.

      so, basically you want to add one array to the end of the other?

      that's still fairly easy

      Code Snippet

      int[][] join = new int[a0.Length + a1.Length][];
      int i = 0;
      for( ; i < a0.Length; i++ )
      {

      join[i] = new int[ a0[1].Length ];

      Array.Copy( a0[i], join[i], a0[i].Length );

      }

      for( int k = 0 ; k < a1.Length; i++, k++ )
      {
      join[i] = new int[ a1[k].Length ];

      Array.Copy( a1[k], join[i], a1[k].Length );

      }

      return join;


      There are some ways to make this more efficient, especially if you aren't concerned about getting copies of the array and just using the arrays as it stands.

      join[ i ] = new int[ a1[k].Length ];

      Array.Copy( a1[k], join[ i ], a1[k].Length );

      easily becomes

      join[ i ] = a1[k];

      however this means that join[ i ] and a1[k] point to the same array in memory, because arrays are reference types.
